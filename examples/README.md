# examples/

Working, runnable demonstrations — distinct from documentation prose in
`docs/`. Every example must be safely deletable without breaking a build;
nothing in the core system may depend on anything here.

## Subdirectories

- `workflow-templates/` — sample workflow template definitions.
- `sample-plugins/` — sample plugins built from `templates/plugin-template`.

## Status

Empty. Examples will be added once there is a real workflow engine and
plugin SDK to demonstrate — an example that doesn't run is worse than no
example at all.
